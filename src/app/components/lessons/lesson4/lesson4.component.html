Lession 4: Services

Services allow us to centralize tasks, manage data in a central place so we don’t have to repeat the same code in multiple components. A service is typically a class with a narrow, well-defined purpose. Instances of services shouldn’t be created manually but by Angular’s dependency injector which automatically injects instances of our services as dependencies to our components - all we need to do is inform Angular that we require such an instance. We do that by:

  • adding a constructor to the component class (and importing the service in the same class) - so Angular knows what service we want
  • providing that service (telling Angular how to create it) in the providers property of the class’ @Component decorator or by using @Injectable with providedIn property on the service class. Always decorating with @Injectable is considered a good practice that improves readability of the code.

    The only difference between these ways of providing is how many instances will be created by the providers when a service is added to the component. @Injectable with ProvidedByRoot will create only one instance, and ProvidedByComponent will create a new instance for every component.
    • Important : The dependency injector is a hierarhical injector: the service is always provided in the chosen component class and all its child components! The highest possible level where we can provide a service is the AppModule, and that means the service is available in all our components.
